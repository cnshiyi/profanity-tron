# v1.0.12 - Single-target suffix coarse filter

Date: 2026-06-21

## 中文

### 目标

减少单目标后 3-12 位匹配时的 Base58 尾部计算开销，为随机模式和指定位数模式同时提速。

### 修改

- 在 OpenCL 匹配核中为单目标后缀 3-12 位增加后 2 位粗筛。
- 候选地址先计算后 2 个 Base58 字符；不匹配时直接返回。
- 只有后 2 位匹配的候选才继续计算后 3-12 位。
- 未修改 UI、范围参数和 NVIDIA 默认调参分档。

### 验证

构建：

- `powershell -NoProfile -ExecutionPolicy Bypass -File .\scripts\build-windows.ps1 -Version v1.0.12-test`，退出码 0。
- 未配置证书时日志显示 `Code signing skipped: no certificate configured.`

同口径 RTX 3070 10 秒测试，`--work 32 --inverse-multiple 196608`：

- v1.0.11 基线随机后 8 位：338.995 MH/s。
- v1.0.12 粗筛后随机后 8 位：354.943 MH/s。
- v1.0.12 粗筛后固定后 16 位向上、地址后 8 位：357.349 MH/s。

正确性验证：

- 初始私钥全 0，后 16 位向上，目标地址 `TJSWuhAyiDQyb7E8ne1AitDV62H8HXRnvU` 后 8 位，`quit-count 1`。
- 输出私钥 `0000000000000000000000000000000000000000000000000000000002fcf6a3`。

额外扫描：

- `--inverse-size 127`：354.708 MH/s。
- `--inverse-size 191`：354.119 MH/s。
- `--inverse-size 255`：356.212 MH/s。
- `--inverse-size 383`：351.556 MH/s。
- `--inverse-size 511`：348.737 MH/s。

测试后未发现残留 `shiyi`、旧 `profanity*`、`TronStudio` 或 `start` 进程。

### 状态

400 MH/s 目标尚未达成。后缀粗筛已经把后 8 位匹配拉近到 suffix 1/2 的上限，下一轮需要继续优化 inverse/iterate 主内核。

## English

### Goal

Reduce Base58 tail computation cost for single-target 3-12 character suffix matching, improving both random and fixed-digit modes.

### Changes

- Added a last-2-character coarse filter in the OpenCL matching kernel for single-target suffix lengths from 3 to 12.
- Candidate addresses compute the last two Base58 characters first and return immediately on mismatch.
- Only candidates matching the last two characters continue to compute suffix positions 3 through 12.
- UI, range arguments, and NVIDIA default tuning tiers were not changed.

### Verification

Build:

- `powershell -NoProfile -ExecutionPolicy Bypass -File .\scripts\build-windows.ps1 -Version v1.0.12-test`, exit code 0.
- Without a configured certificate, the log prints `Code signing skipped: no certificate configured.`

Same-method RTX 3070 10-second tests, `--work 32 --inverse-multiple 196608`:

- v1.0.11 baseline random last-8: 338.995 MH/s.
- v1.0.12 coarse-filter random last-8: 354.943 MH/s.
- v1.0.12 coarse-filter last-16 upward range mode, address last-8: 357.349 MH/s.

Correctness verification:

- All-zero initial key, last-16 upward, target address `TJSWuhAyiDQyb7E8ne1AitDV62H8HXRnvU` last-8, `quit-count 1`.
- Output private key `0000000000000000000000000000000000000000000000000000000002fcf6a3`.

Additional scan:

- `--inverse-size 127`: 354.708 MH/s.
- `--inverse-size 191`: 354.119 MH/s.
- `--inverse-size 255`: 356.212 MH/s.
- `--inverse-size 383`: 351.556 MH/s.
- `--inverse-size 511`: 348.737 MH/s.

No leftover `shiyi`, old `profanity*`, `TronStudio`, or `start` processes were found after tests.

### Status

The 400 MH/s target has not been reached. The suffix coarse filter brings last-8 matching close to the suffix 1/2 ceiling; the next pass needs deeper inverse/iterate kernel optimization.
