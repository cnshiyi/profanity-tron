# v1.0.11 - RTX 3070 default tuning pass

Date: 2026-06-21

## 中文

### 目标

基于串行参数矩阵，把 RTX 3070 这类 64 CU 以下 NVIDIA 显卡的默认参数调到当前实测更合适的组合，同时不把更高端 NVIDIA 显卡固定成 RTX 3070 参数。

### 修改

- 64 CU 以下 NVIDIA 默认 `work` 从 `64` 调为 `32`。
- 64 CU 以下 NVIDIA 默认 `inverse-multiple` 档位从 `131072` 调为 `196608`。
- 64 CU、96 CU、128 CU 以上 NVIDIA 档位仍保留更高批量分档。
- 保留 v1.0.10 的可选 Authenticode 签名构建通道。

### 验证

参数矩阵均串行运行，避免 benchmark 脚本互相清理进程导致假结果。

调参依据样本：

- `--work 64 --inverse-multiple 65536`：351.864 MH/s
- `--work 64 --inverse-multiple 131072`：351.100 MH/s
- `--work 64 --inverse-multiple 196608`：352.579 MH/s
- `--work 64 --inverse-multiple 262144`：215.683 MH/s
- `--work 32 --inverse-multiple 196608`：358.884 MH/s
- `--work 64 --inverse-multiple 196608`：352.326 MH/s
- `--work 128 --inverse-multiple 196608`：348.659 MH/s
- `--work 256 --inverse-multiple 196608`：349.995 MH/s

正式 `v1.0.11` 构建验证：

- 构建命令 `powershell -NoProfile -ExecutionPolicy Bypass -File .\scripts\build-windows.ps1 -Version v1.0.11`，退出码 0。
- 未配置证书时日志显示 `Code signing skipped: no certificate configured.`
- 发布包为 `dist/shiyi-v1.0.11.zip`，包含 `shiyi.exe`、`start.exe`、`start.bat`、`kernels/*.cl`、默认目标和版本文件。
- 默认参数输出：`work = 32`，`inverse-multiple = 196608`，`work-max = 50135040`。
- 随机单目标 6 秒：338.415 MH/s。
- 全 0 初始私钥、后 16 位向上、后 8 位地址测试 6 秒：338.758 MH/s。
- 构建和测试后未发现残留 `shiyi`、旧 `profanity*`、`TronStudio` 或 `start` 进程。

### 状态

400 MH/s 目标尚未达成。v1.0.11 只是默认参数小幅调优和可回溯发版点，下一轮仍需继续做 GPU 粗筛、CPU 细筛或更深的内核路径优化。

## English

### Goal

Use serial parameter-matrix evidence to tune defaults for RTX 3070-class NVIDIA GPUs below 64 compute units, without pinning higher-end NVIDIA GPUs to the same values.

### Changes

- Changed default NVIDIA `work` below 64 compute units from `64` to `32`.
- Changed the same NVIDIA tier default `inverse-multiple` from `131072` to `196608`.
- Kept higher batch tiers for NVIDIA GPUs at 64, 96, and 128+ compute units.
- Kept the optional Authenticode signing build path from v1.0.10.

### Verification

Parameter matrix tests were run serially to avoid false results from benchmark process cleanup.

Tuning evidence:

- `--work 64 --inverse-multiple 65536`: 351.864 MH/s
- `--work 64 --inverse-multiple 131072`: 351.100 MH/s
- `--work 64 --inverse-multiple 196608`: 352.579 MH/s
- `--work 64 --inverse-multiple 262144`: 215.683 MH/s
- `--work 32 --inverse-multiple 196608`: 358.884 MH/s
- `--work 64 --inverse-multiple 196608`: 352.326 MH/s
- `--work 128 --inverse-multiple 196608`: 348.659 MH/s
- `--work 256 --inverse-multiple 196608`: 349.995 MH/s

Official `v1.0.11` build verification:

- Build command `powershell -NoProfile -ExecutionPolicy Bypass -File .\scripts\build-windows.ps1 -Version v1.0.11`, exit code 0.
- Without a configured certificate, the log prints `Code signing skipped: no certificate configured.`
- Release package is `dist/shiyi-v1.0.11.zip`, containing `shiyi.exe`, `start.exe`, `start.bat`, `kernels/*.cl`, default targets, and version metadata.
- Default tuning output: `work = 32`, `inverse-multiple = 196608`, `work-max = 50135040`.
- Random single-target, 6 seconds: 338.415 MH/s.
- All-zero initial key, last-16 upward, address last-8 test, 6 seconds: 338.758 MH/s.
- No leftover `shiyi`, old `profanity*`, `TronStudio`, or `start` processes were found after build and tests.

### Status

The 400 MH/s target has not been reached. v1.0.11 is only a small default-parameter tuning and traceable release point; the next pass still needs GPU coarse filtering, CPU fine filtering, or deeper kernel-path optimization.
