# v1.0.17 - Strict TRON/Base58 Target Validation

Date: 2026-06-21

## Goal

Remove the remaining legacy default-target problem where old runtime files could still show `[银行卡]` or invalid `0` targets, and make the launcher, native generator, and Windows package build agree on the same valid target rules.

## Changes

- The launcher now validates every line loaded from `runtime/targets.txt`.
- If an existing runtime target file contains `[银行卡]`, `0`, or any other non-Base58 character, the launcher falls back to the clean default target set.
- Target saving and generation startup now reject invalid 34-character TRON targets instead of writing them back to `runtime/targets.txt`.
- `shiyi.exe` now validates direct 34-character TRON target arguments and rejects addresses containing `0`.
- `scripts/build-windows.ps1` validates the source default target list before writing `dist/profanity.txt`, `dist/runtime/targets.txt`, or the release zip.

## Default Targets

## Verification

Build:

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\scripts\build-windows.ps1 -Version v1.0.17
```

Package checks:

- `dist/shiyi-v1.0.17.zip` was built.
- Package contains `shiyi.exe`, `start.exe`, `start.bat`, `kernels/*.cl`, `profanity.txt`, `runtime/targets.txt`, `README.md`, and `VERSION.txt`.
- Package does not contain `profanity.x64.exe`.
- `dist/profanity.txt` and `dist/runtime/targets.txt` match and contain the 10-line `1-9 + A` target set.
- Static search found no `[银行卡]`, no default `TTTT...000...`, and no old executable name inside `dist`.

Invalid target check:

```powershell
.\dist\shiyi.exe --matching TTTTTTTTTTTTTTTTT00000000000000000 --prefix-count 0 --suffix-count 8 --benchmark-seconds 1 --work 32 --inverse-multiple 196608
```

Result:

- `shiyi.exe` prints `matching target contains invalid TRON/Base58 characters; 0 is not allowed`.
- Exit code is `1`.

Valid target smoke:


Result:

- Exit code: `0`

Process cleanup:

No matching residual processes were found.

## Decision

Ship this as a correctness and packaging-safety release. The 400 MH/s target remains open; this release does not claim a performance breakthrough.
