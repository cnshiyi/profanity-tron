# v1.0.13 - Multi-target suffix coarse filter

Date: 2026-06-21

## 中文

### 目标

提升默认 10 目标列表的速度，特别是目标中包含 Base58 非法字符 `0` 时，避免这些不可能命中的目标拖慢多目标后缀匹配路径。

### 修改

- 多目标后缀 2-12 位统一先计算后 2 位 Base58 索引。
- 主机端为所有可解析的目标登记可命中的后两位组合。
- GPU 端如果后两位不在任何目标组合中，直接返回。
- 含 Base58 非法字符的目标不会登记组合，因此不会触发完整尾部比较。
- 保留 v1.0.12 的单目标后缀粗筛。

### 验证

构建：

- `powershell -NoProfile -ExecutionPolicy Bypass -File .\scripts\build-windows.ps1 -Version v1.0.13-test`，退出码 0。
- 未配置证书时日志显示 `Code signing skipped: no certificate configured.`

RTX 3070，同口径 `--work 32 --inverse-multiple 196608`：

- 默认 10 目标列表 8 秒：348.527 MH/s。
- v1.0.12 默认 10 目标列表基线：328.545 MH/s。
- 合法 Base58 10 目标列表 8 秒：354.192 MH/s。
- 随机单目标后 8 位 8 秒：356.208 MH/s。
- 全 0 初始私钥、后 16 位向上、地址后 8 位 10 秒：356.719 MH/s。

正确性验证：

- 目标地址 `TJSWuhAyiDQyb7E8ne1AitDV62H8HXRnvU` 后 8 位，`quit-count 1`。
- 输出私钥 `0000000000000000000000000000000000000000000000000000000002fcf6a3`。

profiling 观察：

- `profanity_iterate` 约占 50%。
- `profanity_inverse` 约占 31%。
- `profanity_score` 约占 19%。
- 读回几乎为 0。

测试后未发现残留 `shiyi`、旧 `profanity*`、`TronStudio` 或 `start` 进程。

### 状态

400 MH/s 目标尚未达成。下一轮需要继续优化 inverse/iterate 主内核和 checksum score 路径。

## English

### Goal

Improve the default 10-target list speed, especially when targets contain the non-Base58 character `0`, so impossible targets no longer slow down the multi-target suffix path.

### Changes

- Multi-target suffix matching from 2 to 12 chars now computes the last two Base58 indices first.
- The host registers possible last-two-character combinations for every parseable target.
- The GPU returns immediately when the candidate's last two chars are not in any registered target combination.
- Targets containing non-Base58 characters do not register combinations, so they do not trigger full-tail comparisons.
- Keeps the v1.0.12 single-target suffix coarse filter.

### Verification

Build:

- `powershell -NoProfile -ExecutionPolicy Bypass -File .\scripts\build-windows.ps1 -Version v1.0.13-test`, exit code 0.
- Without a configured certificate, the log prints `Code signing skipped: no certificate configured.`

RTX 3070, same-method `--work 32 --inverse-multiple 196608`:

- Default 10-target list, 8 seconds: 348.527 MH/s.
- v1.0.12 default 10-target baseline: 328.545 MH/s.
- Valid Base58 10-target list, 8 seconds: 354.192 MH/s.
- Random single-target last-8, 8 seconds: 356.208 MH/s.
- All-zero initial key, last-16 upward, address last-8, 10 seconds: 356.719 MH/s.

Correctness verification:

- Target address `TJSWuhAyiDQyb7E8ne1AitDV62H8HXRnvU` last-8, `quit-count 1`.
- Output private key `0000000000000000000000000000000000000000000000000000000002fcf6a3`.

Profiling observation:

- `profanity_iterate` is about 50%.
- `profanity_inverse` is about 31%.
- `profanity_score` is about 19%.
- Readback is effectively zero.

No leftover `shiyi`, old `profanity*`, `TronStudio`, or `start` processes were found after tests.

### Status

The 400 MH/s target has not been reached. The next pass needs deeper inverse/iterate kernel and checksum score-path optimization.
