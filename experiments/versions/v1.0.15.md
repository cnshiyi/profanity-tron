# v1.0.15 - Duplicate Counting And Target Cleanup

Date: 2026-06-21

## Goal

Continue the optimization loop toward 400 MH/s while fixing correctness issues reported from launcher runs:

- default targets still contained an impossible `0` target
- range-mode hit output could repeatedly count the same private key
- `--quit-count` behaved like a score threshold instead of a hit count
- long benchmark runs needed reliable timeout evidence

This release does not claim a 400 MH/s performance breakthrough.

## Changes

- Default targets now remove the invalid Base58 character `0`; the default 10-line set is `1-9 + A`.
- `profanity.txt`, `dist/profanity.txt`, and `dist/runtime/targets.txt` are synchronized.
- The GPU result buffer is cleared before every scoring round.
- The OpenCL result path now uses slot `0` as a hit counter and slots `1..120` as per-batch hit records.
- CPU result processing deduplicates by private key before increasing validated/saved hit counts.
- `--quit-count` now means unique validated hit count, so launcher count `999` no longer overflows a `cl_uchar` score threshold.
- `scripts/benchmark-local.ps1` uses async `ReadToEndAsync()` pipes and returns structured `BLOCKED` / `TIMEOUT` evidence with stdout/stderr files.

## Blocker

After the final rebuilt native binary, local execution was blocked by Smart App Control / Code Integrity:

```text
An Application Control policy has blocked this file
```

Windows Code Integrity event `3077` reported that `dist\shiyi.exe` did not meet the enterprise signing level requirements.

Because the system policy blocked the final local binary, the requested 1-minute random and range performance tests could not be completed on this machine. This release therefore does not claim that random or fixed-digit range mode reaches 400 MH/s.

## Reference Performance

Official v1.0.14 release package reference retests on RTX 3070:

- Random single-target last-8, 20 seconds: 357.353 MH/s
- All-zero initial key, last-16 upward range, address last-8, 20 seconds: 356.712 MH/s

Rejected temporary kernel experiments:

- Split modular-multiply reduction branch: 310.538 MH/s random last-8 over 6 seconds.
- `modHigher` constant-memory experiment: 357.988 MH/s random last-8 over 6 seconds.

## Process Cleanup

Checked after tests:

```powershell
Get-Process shiyi,profanity*,TronStudio*,start -ErrorAction SilentlyContinue
```

No matching residual processes were found.

## Decision

Ship the correctness and diagnostics fixes, but keep the 400 MH/s target open. Future performance claims require 1-minute verification on a signed or policy-allowed build.
