# v1.0.16 - Profiling Support And 1-Minute Baselines

Date: 2026-06-21

## Goal

Continue the optimization loop toward 400 MH/s for both random and fixed-digit range generation, while keeping every experiment traceable and avoiding unproven performance claims.

## Changes

- Added `scripts/build-windows.ps1 -DebugNative`.
- `-DebugNative` compiles the native generator with `PROFANITY_DEBUG` for OpenCL profiling.
- Debug profiling now reports inverse, iterate, and score kernel timings.
- The default release build remains unchanged unless `-DebugNative` is explicitly passed.

## Verification

Online v1.0.15 release package 1-minute serial baselines:

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\scripts\benchmark-local.ps1 -ExePath .\experiments\tmp\release-v115-run\shiyi.exe -Name release_v115_random_60s_serial -ArgumentLine "--matching TTTTTTTTTTTTTTTTT99999999999999999 --prefix-count 0 --suffix-count 8 --benchmark-seconds 60 --work 32 --inverse-multiple 196608" -TimeoutSeconds 300
```

- Random last-8: 383.908 MH/s


Profiling sample, excluding the first warmup sample:

- score average: 24.7 ms

## Rejected Experiments

- Benchmark mode skipping result-buffer clearing: 60-second random remained 383.765 MH/s, range remained 383.178 MH/s.
- Partial SHA256 message-schedule clearing: 8-second serial random was 382.882 MH/s and range was 384.338 MH/s.
- OpenCL build options `-cl-mad-enable -cl-no-signed-zeros`: 60-second random remained 383.763 MH/s.
- `negativeGx` constant-memory experiment: 60-second random remained 383.924 MH/s.

No rejected experiment is shipped as a performance optimization.

## Process Cleanup

Checked after tests:

```powershell
Get-Process shiyi,profanity*,TronStudio*,start -ErrorAction SilentlyContinue
```

No matching residual processes were found.

## Decision

Ship profiling support and updated evidence. The 400 MH/s target remains open; the next optimization pass should prioritize the iterate and inverse kernels.
