# v1.0.7 - target-line cleanup and README encoding

Date: 2026-06-21

## 中文

### 目标

修复目标地址文件的“一行一个”可靠性，避免首行被 UTF-8 BOM 隐藏字符污染；同时修复发布包 README 中文在 PowerShell 5.1 构建时乱码的问题。

### 修改

- 启动器保存 `runtime/targets.txt` 时使用 UTF-8 无 BOM。
- 启动器读取/保存目标地址时清理空行、首尾空白和 BOM。
- 原生生成器 `Mode::matching` 读取文件或单个目标时清理 BOM 和首尾空白，兼容旧文件。
- `scripts/build-windows.ps1` 生成默认目标文件时使用 UTF-8 无 BOM，并保证每个目标独立成行。
- 发布包 README 改为从 `docs/package-README.md` UTF-8 模板复制，避免脚本源码编码导致中文乱码。

### 验证

- 构建 `dist/shiyi-v1.0.7.zip`。
- 检查 `dist/runtime/targets.txt` 第一字节为 `54 54 54`，不是 UTF-8 BOM。
- 检查目标文件共有 10 行，每行一个目标。
- 使用带 BOM 的临时目标文件运行 `shiyi.exe`，`ExitCode: 0`，速度样本 `410.794 MH/s`。
- 检查发布包 README 包含正常中文和 English 段落。
- 启动器烟测：可启动和关闭，退出后无 `shiyi`、旧 `profanity*`、`TronStudio` 或 `start` 残留进程。

## English

### Goal

Fix target-list one-per-line reliability by preventing hidden UTF-8 BOM contamination on the first line, and fix packaged README Chinese text when building with Windows PowerShell 5.1.

### Changes

- The launcher writes `runtime/targets.txt` as UTF-8 without BOM.
- The launcher cleans blank lines, surrounding whitespace, and BOM when loading/saving targets.
- Native `Mode::matching` strips BOM and surrounding whitespace for both file-based and single-target matching.
- `scripts/build-windows.ps1` writes default target files as UTF-8 without BOM and keeps one target per line.
- The packaged README is copied from `docs/package-README.md`, a UTF-8 template, instead of embedding Chinese text inside the PowerShell script.

### Verification

- Build `dist/shiyi-v1.0.7.zip`.
- Check that `dist/runtime/targets.txt` starts with `54 54 54`, not a UTF-8 BOM.
- Check the target file has 10 lines, one target per line.
- Run `shiyi.exe` with a temporary BOM-prefixed target file: `ExitCode: 0`, speed sample `410.794 MH/s`.
- Check the packaged README contains normal Chinese and English sections.
- Launcher smoke test: start and close; no leftover `shiyi`, old `profanity*`, `TronStudio`, or `start` processes after exit.
