# v1.0.9 - last-lane range correctness and adaptive tuning

Date: 2026-06-21

## 中文

### 目标

修复“初始私钥全 0、后 16 位向上、生成地址后 8 位”这一指定范围场景，并为更高端 NVIDIA 显卡保留自适应调参空间。

### 修改

- 范围模式从只支持私钥前 16 位，改为支持任意一个 16 位私钥块内的连续 1-16 个十六进制位。
- 启动器位数范围从修改前 16 位改为修改私钥后缀位数。
- 全 0 初始私钥自动跳过无效私钥 `0`，从 `...0001` 开始扫描。
- 修复范围模式命中私钥回写少 1 的问题。
- 命令行相对 `--output` 路径会自动创建父目录。
- NVIDIA 默认调参按显存预算和计算单元分档，不再固定到 RTX 3070 一组参数。

### 验证

- 构建：`powershell -NoProfile -ExecutionPolicy Bypass -File .\scripts\build-windows.ps1 -Version v1.0.9-test`
- `shiyi.exe --help` 正常运行。
- 全 0 初始私钥、后 16 位向上、地址后 8 位：输出 `variable hex = 49-64`、`lane = 0`，并提示跳过无效私钥 `0`。
- 低难度命中回归：地址 `TMy8JcLsT3cAwTMdNbEBHY4QPoJ1mdrUyK`，私钥 `000...008b`，独立 secp256k1 + Keccak 校验通过。
- 相对输出路径 `experiments\tmp\relative_output_hit.txt` 保存成功。
- 测试后无 `shiyi`、旧 `profanity*`、`TronStudio` 或 `start` 残留进程。
- 正式 `v1.0.9` 构建包内容已验证；本机再次运行 `dist\shiyi.exe --help` 时被 Windows Application Control 策略拦截，Code Integrity 事件 3033/3077 显示未满足企业签名级别要求。

### 性能

RTX 3070 当前可信样本：

- 随机单目标 5 秒：353.158 MH/s
- 默认多目标 5 秒：331.924 MH/s
- 全 0 初始私钥、后 16 位向上、后 8 位地址测试 5 秒：333.261 MH/s

400 MH/s 目标尚未达成，下一轮继续优化 GPU 粗筛、CPU 细筛和范围模式批次步进。

## English

### Goal

Fix the all-zero initial key, last-16 upward, address last-8 range scenario and keep adaptive tuning room for higher-end NVIDIA GPUs.

### Changes

- Range mode now supports any continuous 1-16 hex chars inside a single 16-char private-key block, instead of only the first 16 hex chars.
- The launcher digit-count range builder now edits private-key suffix digits.
- An all-zero initial private key automatically skips invalid private key `0` and starts at `...0001`.
- Fixed range-mode private-key reporting being off by one.
- Relative command-line `--output` paths create parent directories before saving.
- NVIDIA default tuning now uses memory-budget and compute-unit tiers instead of one RTX 3070-class parameter set.

### Verification

- Build: `powershell -NoProfile -ExecutionPolicy Bypass -File .\scripts\build-windows.ps1 -Version v1.0.9-test`
- `shiyi.exe --help` runs successfully.
- All-zero initial key, last-16 upward, address last-8 test prints `variable hex = 49-64`, `lane = 0`, and reports that invalid private key `0` was skipped.
- Low-difficulty hit regression: address `TMy8JcLsT3cAwTMdNbEBHY4QPoJ1mdrUyK`, private key `000...008b`, independently verified with secp256k1 + Keccak.
- Relative output path `experiments\tmp\relative_output_hit.txt` saves successfully.
- No leftover `shiyi`, old `profanity*`, `TronStudio`, or `start` processes after tests.
- The final `v1.0.9` package contents were verified; rerunning `dist\shiyi.exe --help` on this machine was blocked by Windows Application Control. Code Integrity events 3033/3077 report that the binary did not meet Enterprise signing level requirements.

### Performance

Current trusted RTX 3070 samples:

- Random single-target, 5 seconds: 353.158 MH/s
- Default multi-target, 5 seconds: 331.924 MH/s
- All-zero initial key, last-16 upward, address last-8 test, 5 seconds: 333.261 MH/s

The 400 MH/s target has not been reached yet. Continue with GPU coarse filtering, CPU fine filtering, and range-mode batch stepping in the next pass.
