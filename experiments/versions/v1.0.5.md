# v1.0.5 - range wiring and benchmark hygiene

Date: 2026-06-21

## Goal

Continue the explicit optimization loop toward 400 MH/s for both random and fixed-digit modes.

## Changes

- Restored the range-capable native backend from the recovered v1.0.3 source path.
- Added runtime OpenCL kernel sources to the tracked source tree and release package.
- Wired launcher fields for initial private key, direction, and digit count into `--range-start`, `--range-end`, and `--range-direction`.
- Limited fixed digit count to 1-16 hex digits and added launcher guidance.
- Added a local benchmark helper that runs benchmarks serially and cleans same-folder residual generator processes before each run.
- Reset speed sampling after initialization, so small fixed-digit windows are not reported with initialization time mixed into throughput.
- Shrunk range-mode GPU batch size for small windows, avoiding large unused initialization work.
- Restored the release generator name to `shiyi.exe`; legacy generator names are kept only for cleanup compatibility.
- Removed obsolete static curl/network link dependencies from the Windows native build after Smart App Control blocked the newly built executable.
- Fixed benchmark helper process launching so tests run serially and capture stdout/stderr/exit code without racing against cleanup.

## Benchmarks

Machine: NVIDIA GeForce RTX 3070

Commands:

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\scripts\benchmark-local.ps1 -ExePath .\dist\shiyi.exe -Name random_5s_serial -ArgumentLine '--matching TUqEg3dzVEJNQSVW2HY98z5X8SBdhmao8D --prefix-count 1 --suffix-count 1 --benchmark-seconds 5' -TimeoutSeconds 140
powershell -NoProfile -ExecutionPolicy Bypass -File .\scripts\benchmark-local.ps1 -ExePath .\dist\shiyi.exe -Name default_multi_5s_serial -ArgumentLine '--matching runtime\targets.txt --prefix-count 0 --suffix-count 5 --benchmark-seconds 5' -TimeoutSeconds 140

```

Results:

- Random single-target short run after removing obsolete network link dependencies: 383.894 MH/s.
- Default multi-target short run after removing obsolete network link dependencies: 382.824 MH/s.
- Fixed-digit range before speed reset and batch clamp: 2.561 MH/s, caused by initialization time and oversized batch dominating a 2,097,152-key range.
- Fixed-digit range after fix and rebuild cleanup: 419.475 MH/s.
- `.\dist\shiyi.exe --help` runs without Smart App Control blocking after removing obsolete static curl/network libraries from the native link step.
- Benchmark helper smoke test captured `ExitCode: 0` and `440.799 MH/s` on a 2-second random run.

Residual process checks after benchmark runs found no `shiyi`, old `profanity*`, `TronStudio`, or `start` processes.

## Status

400 MH/s is reached for the explicit initial-key range test, but not yet for random/default modes. This version restores a reliable baseline, makes fixed-digit mode usable, and records the next optimization target.
