# v1.0.20 - Benchmark isolation guard and rejected optimization record

Date: 2026-06-21

## Goal

Continue the explicit optimization loop toward 400 MH/s while keeping every benchmark result reproducible and traceable.

This pass found that `scripts/benchmark-local.ps1` could invalidate parallel verification runs by stopping another benchmark process from the same output directory. That made one 60-second random run exit with code `-1` while a range run was still executing.

## Code Change

- `scripts/benchmark-local.ps1` no longer silently kills already-running `shiyi` or legacy `profanity*` processes in the same executable directory.
- If an existing benchmark target process is found, the script returns a structured `RUNNING` result and explains which process is active.
- Added an explicit `-StopExisting` switch for the rare case where cleanup is intentional.

## Rejected Experiments

These experiments were tested and reverted because they did not produce meaningful 60-second improvement:

- Keccak state initialization shortcut:
  - Replaced full `ethhash` zero initialization with explicit padding/empty-lane initialization.
  - 20-second random result: `371.656 MH/s`.
  - Existing local short baseline: `372.618 MH/s`.
  - Decision: reverted.
- Single-target exact suffix-4 coarse filter:
  - Added `58^4` rolling modulo before full Base58 tail checks.
  - 20-second random result: `370.423 MH/s`.
  - Correctness smoke still hit suffix `999`, but the extra per-candidate modulo cost outweighed the smaller candidate set.
  - Decision: reverted.
- Fused iterate+score kernel:
  - Combined `profanity_iterate` and `profanity_score_matching` into a single experimental kernel while keeping the same hash buffer.
  - 20-second random result: `373.457 MH/s`.
  - 60-second random result: `370.144 MH/s`.
  - 60-second last-16 upward range result: `371.519 MH/s`.
  - Existing local 60-second baseline: random `370.007 MH/s`, range16 `369.738 MH/s`.
  - Decision: reverted because the gain was too small/noisy for the added duplicated kernel surface.
