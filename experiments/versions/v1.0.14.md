# v1.0.14 - Manual batch safety guard

Date: 2026-06-21

## 中文

### 目标

避免用户手动输入过大的 `--inverse-multiple` 导致初始化阶段异常退出，同时补充参数说明。

### 修改

- 当用户手动设置 `--inverse-multiple` 且未设置 `--work-max` 时，按每个 GPU 60% 显存预算计算安全上限。
- 超过安全上限时返回错误，并提示使用安全值或留空自动调参。
- `--help` 增加 `--inverse-multiple` 的安全说明。

### 验证

- 构建命令 `powershell -NoProfile -ExecutionPolicy Bypass -File .\scripts\build-windows.ps1 -Version v1.0.14-test`，退出码 0。
- 未配置证书时日志显示 `Code signing skipped: no certificate configured.`
- `--inverse-multiple 327680` 在 RTX 3070 上返回退出码 1，并提示 `use 210524 or leave it empty for automatic tuning`。
- 默认随机单目标后 8 位 6 秒：356.589 MH/s。
- 全 0 初始私钥、后 16 位向上、后 8 位地址测试 6 秒：356.337 MH/s。
- 测试后未发现残留 `shiyi`、旧 `profanity*`、`TronStudio` 或 `start` 进程。

### 状态

400 MH/s 目标尚未达成。本版本是稳定性和文档补强，下一轮继续优化 inverse/iterate 主内核和 checksum score 路径。

## English

### Goal

Prevent overly large manual `--inverse-multiple` values from crashing during initialization, and document the safer usage.

### Changes

- When the user sets `--inverse-multiple` manually without `--work-max`, the program computes a safe cap from a 60% memory budget for each GPU.
- Values above the cap return a clear error suggesting the safe value or automatic tuning.
- Added `--inverse-multiple` safety notes to `--help`.

### Verification

- Build command `powershell -NoProfile -ExecutionPolicy Bypass -File .\scripts\build-windows.ps1 -Version v1.0.14-test`, exit code 0.
- Without a configured certificate, the log prints `Code signing skipped: no certificate configured.`
- `--inverse-multiple 327680` returns exit code 1 on RTX 3070 and prints `use 210524 or leave it empty for automatic tuning`.
- Default random single-target last-8, 6 seconds: 356.589 MH/s.
- All-zero initial key, last-16 upward, address last-8 test, 6 seconds: 356.337 MH/s.
- No leftover `shiyi`, old `profanity*`, `TronStudio`, or `start` processes were found after tests.

### Status

The 400 MH/s target has not been reached. This version is a stability and documentation pass; the next pass continues with inverse/iterate kernel and checksum score-path optimization.
