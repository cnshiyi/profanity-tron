# v1.0.8 - default target correction

Date: 2026-06-21

## 中文

### 目标

修正默认目标地址列表，去掉误写入的旧占位文本，恢复为数字目标集合。

### 修改

- 根目录 `profanity.txt` 改为 10 行默认目标：`000...` 到 `999...`。
- Windows 启动器内置兜底默认值同步改为同一组 10 行目标。
- 启动器读取已有 `runtime/targets.txt` 时，如果发现旧占位文本，会自动回退到新的默认目标，避免旧运行目录把错误内容带回来。
- 发布包中的 `profanity.txt` 和 `runtime/targets.txt` 均由构建脚本重新生成。

### 验证

- 源文件、`dist/profanity.txt`、`dist/runtime/targets.txt` 均为 10 行。
- 三份目标文件内容完全一致。
- 三份目标文件首字节均为 `54 54 54`，无 UTF-8 BOM。
- 发布 zip 内的 `profanity.txt` 和 `runtime/targets.txt` 均为 0-9 十行目标。

### 性能状态

本版本不发布未验证的性能参数调整。随机/default 模式超过 400 MH/s 的目标仍继续迭代。

## English

### Goal

Correct the default target list by removing the accidental legacy placeholder text and restoring digit target lines.

### Changes

- Changed root `profanity.txt` to the 10-line default target set: `000...` through `999...`.
- Updated the Windows launcher fallback defaults to the same 10-line target set.
- When the launcher reads an existing `runtime/targets.txt`, it falls back to the new defaults if legacy placeholder text is detected, preventing old runtime directories from reintroducing the bad targets.
- The release package `profanity.txt` and `runtime/targets.txt` are regenerated by the build script.

### Verification

- Source file, `dist/profanity.txt`, and `dist/runtime/targets.txt` all contain 10 lines.
- All three target files match exactly.
- All three target files start with bytes `54 54 54`, with no UTF-8 BOM.
- The release zip contains the corrected 0-9 target list in both `profanity.txt` and `runtime/targets.txt`.

### Performance Status

This version does not publish the unverified performance parameter experiment. The random/default mode target of more than 400 MH/s remains active for the next optimization pass.
